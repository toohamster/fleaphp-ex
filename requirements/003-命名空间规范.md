# 需求分析：命名空间和全局函数调用规范

## 问题来源

在修复 `sys_get_tempdir()` 函数名错误时，发现对 PHP 命名空间函数解析规则理解不准确。

## 问题描述

### 错误案例

```php
namespace FLEA\Config;

// 错误：使用了不存在的函数
'internalCacheDir' => env('CACHE_DIR', sys_get_tempdir() . DS . 'fleaphp_cache');
//                                      ^^^^^^^^^^^^^^^^ 这个函数在 PHP 7.4 不存在
```

### 正确的理解

1. **PHP 命名空间函数解析规则**：
   - PHP 先在当前命名空间查找函数 `FLEA\Config\sys_get_temp_dir()`
   - 找不到时自动回退到全局命名空间 `\sys_get_temp_dir()`
   - 无论直接调用还是作为参数传递，都**不需要加 `\`**

2. **常量的命名空间规则**：
   - 常量在命名空间中查找时**不会**自动回退到全局命名空间
   - 全局常量必须使用 `\DS` 或完全限定名

## 整改方案

### 规则 1：使用全局函数

```php
namespace FLEA\Config;

// ✓ 正确 - 自动回退到全局命名空间
$value = sys_get_temp_dir();
$value = strtoupper('test');
$result = array_merge($a, $b);

// × 错误 - 冗余但不错
$value = \sys_get_temp_dir();
```

### 规则 2：使用全局常量

```php
namespace FLEA\Config;

// ✓ 正确 - 显式引用全局常量
$path = sys_get_temp_dir() . \DS . 'cache';

// × 错误 - 会在当前命名空间查找 DS
$path = sys_get_temp_dir() . DS . 'cache';
```

### 规则 3：PHP 版本兼容性检查

在使用任何 PHP 函数前，必须：
1. 用 `php74 -r "function_exists('xxx')"` 验证
2. 或查阅 PHP 官方文档确认函数引入版本
3. 不应凭记忆写代码
