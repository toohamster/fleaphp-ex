# Claude AI 工具使用规范

## 编辑文件
- Edit 工具只试 1 次，匹配失败立即换 Write 或 Bash sed
- 删除大段代码（超过 20 行）用 Bash sed 或 Write 重写整个文件
- 使用 Edit 时必须完全匹配原文本（包括空格、换行、标点）

## 删除文件
- 删除文件中特定行：使用 Bash sed 命令 `sed -i.bak 'N,Md' filename`
- 删除废弃文件：直接使用 Bash rm 命令

## 代码规范
- 不擅自回退用户代码，需要用户授权
- 不添加用户未要求的功能或"改进"
- 只对用户明确要求的代码进行修改

## 历史教训
1. **错误回退用户代码** - 曾使用 git checkout -- 回退用户未提交的重构代码
2. **删除函数浪费 10 分钟** - 应直接使用 sed 或 Write 重写，而非反复尝试 Edit
3. **污染项目文档** - 曾将工具规范添加到 CLAUDE.md，应保持项目文档简洁
4. **记录位置错误** - 曾将工具规范记入系统记忆目录，应记在项目 .claude/memory/memory.md
5. **浪费时间在小事情上** - 目录规划这种事应该一次听懂直接执行，不应该反复纠正浪费用户时间
6. **不要再浪费时间** - 用户明确要求后不应再出现反复确认、边做边问的情况
7. **修改代码前必须先给方案，等用户确认后再执行** - 曾擅自修改 Router.php，反复纠正多次才理解正确方案，浪费大量用户时间
